Reward Structure

Fair compensation based on vulnerability severity and potential impact. All public submissions are eligible for rewards.

Critical

$2,500 - $5,000

Remote code execution, authentication bypass, complete system compromise

Examples:

  • Encryption key extraction
  • Authentication bypass
  • Remote code execution
  • Complete privacy breach
High

$500 - $2,000

Significant security impact, partial system compromise, cryptographic flaws

Examples:

  • Message decryption flaws
  • Key exchange vulnerabilities
  • Privilege escalation
  • SQL injection
Medium

$200 - $500

Moderate security impact, information disclosure, limited compromise

Examples:

  • Cross-site scripting (XSS)
  • Information disclosure
  • CSRF vulnerabilities
  • Metadata leakage
Low

$50 - $200

Minor security issues, configuration problems, low-impact vulnerabilities

Examples:

  • Security misconfigurations
  • Missing security headers
  • Minor information leaks
  • Rate limiting bypass

Program Rules & Public Policy

Clear guidelines for public vulnerability disclosure and bug bounty participation.

‚úÖ In Scope

üîê Enclave Messenger Applications

  • Desktop GUI application (Python/Tkinter)
  • Command-line interface (CLI)
  • Web interface (Flask application)
  • Core encryption module (secure_messenger.py)

üåê Web Properties

  • Official website: enclave-messenger.com
  • Documentation site: docs.enclave-messenger.com
  • Bug bounty portal (this site)
  • GitHub repositories: github.com/Enclave-Software/*

üîß Infrastructure

  • Network protocols and implementations
  • Key exchange mechanisms
  • Database security (SQLite)
  • File system operations

‚ùå Out of Scope

  • Social engineering attacks against employees or users
  • Physical security issues requiring device access
  • Denial of Service (DoS) attacks
  • Third-party services and dependencies
  • Issues requiring significant social engineering
  • Theoretical vulnerabilities without practical impact
  • Spam or content injection without security implications
  • Rate limiting on non-security critical endpoints
  • Missing HTTP headers without demonstrated security impact
  • Issues already known or reported by other researchers

üìã Public Disclosure Policy

1. Submit Through Official Channels

Use our public form above or email security@enclave-messenger.com

2. No Public Disclosure (Initially)

DO NOT create public GitHub issues, blog posts, or social media posts about the vulnerability before we've had time to fix it

3. Acknowledgment Timeline

We acknowledge all public submissions within 48 hours with a tracking ID

4. Fix & Reward Timeline

  • Critical: Fixed within 24 hours, reward within 7 days
  • High: Fixed within 1 week, reward within 14 days
  • Medium: Fixed in next release, reward within 30 days
  • Low: Fixed when possible, reward within 45 days

5. Public Recognition

With your consent, you'll be added to our public Hall of Fame after the fix is deployed

‚öñÔ∏è Legal Safe Harbor

We provide legal safe harbor for security researchers who:

  • Submit vulnerabilities through our official public channels
  • Only test against their own systems or with explicit permission
  • Do not access, modify, or delete data belonging to others
  • Do not perform testing that harms system availability
  • Follow responsible disclosure practices
  • Act in good faith to improve security

We will not pursue legal action against researchers who follow these guidelines.

Public Repository Setup

# Clone the public repository
git clone https://github.com/Enclave-Software/Enclave-Messenger.git
cd Enclave-Messenger

# Install dependencies
python3 setup.py

# Run different interfaces
python3 enclave_messenger_gui.py    # GUI version
python3 enclave_messenger_cli.py    # CLI version  
python3 enclave_messenger_web.py    # Web version

Testing Guidelines

  • Test only on your own local installations
  • Use isolated test environments
  • Default port: 12345 (configurable)
  • All testing should be done offline or in controlled networks

üìù Public Reporting Best Practices

Clear Title

Use descriptive titles like "SQL Injection in User Search" rather than "Security Bug"

Detailed Steps

Provide step-by-step reproduction instructions that our team can follow exactly

Impact Assessment

Explain what an attacker could achieve and the potential business/user impact

Proof of Concept

Include screenshots, code snippets, or demo videos when possible

Environment Details

Specify OS, application version, and other relevant environmental factors
